# simpleGitFlow

这是一个简化版本的[gitflow](https://github.com/nvie/gitflow.git)。它实现了我趣在生产活动中使用的git工作模型。![woqu-gitflow](https://raw.githubusercontent.com/stirp/simpleGitFlow/master/woqu-gitflow.jpg)

这个工作模型需要四个分支类型和三套环境：

* master 分支：用于生产和预生产环境。当代码需要部署到生产环境，它首先需要合并到master，编译并部署到预上线。当预上线的测试结束后，编译后的代码会直接拷贝到生产环境。在开发和部署非常频繁的流程里，是非常难定义一个里程碑和发布分支的，因此我们并不需要这类分支。

* develop 分支, 用于功能测试。所有代码在合并到master之前都需要在这个分支集成并测试。与它相配合的是测试环境，使用的持久层与生产、预生产环境要区别开。

* feature 分支, 大多数只存在于开发者的硬盘。只有那些需要合作的功能才会发布到远端仓库。

* hotfix 分支，如果生产环境的执行情况与预期不符，就需要从之前的tag里创建一个hotfix 分支。这个分支的代码跟生产环境的代码是一致的。hotfix分支也需要到测试环境测试后才能发布。

* 生产环境。

* 预生产环境，这个环境的代码需要良好测试后再发布到生产环境。跟生产环境使用同一个数据源。

* TEST environment, for testing purpose.

Commands:
* sgflow init, make develop branch avalable
* sgflow feature start $name, create a $name branch
* sgflow feature test, merge current branch into develop and push to remote repository
* sgflow feature online, 
* sgflow hotfix start [$tag_name] $name, create a $name branch from $tag_name. If not specified which tag, the latest tag will be used. 
* sgflow hotfix test, merge current branch into develop and push to remote origin
* sgflow hotfix online, merge current branch into master and push to remote origin
* sgflow update, git merge origin master into current branch.
